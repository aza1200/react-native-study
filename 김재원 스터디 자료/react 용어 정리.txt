           서버에서 JavaScript가 작동되도록 해주는 런타임 환경(플랫폼)

Java(*쓰레드 기반 동기방식)와 달리 **단일쓰레드 기반 비동기방식
*:하나의 쓰레드가 request를 받으면 모든 처리가 완료될때까지 기다리다가 처리결과가 완료되면 다시 응답을 보냄
  기존 업무 처리가 완료되기 전에 또다른 request가 있으면 새로운 쓰레드가 업무를 처리함.
  동시 request가 많은 경우 많은 쓰레드가 필요하게 되어 서버 과부하
**:하나의 쓰레드가 request를 받으면 바로 다음 처리에 요청을 보내놓고 다른 작업을 처리하다가 먼저 요청한 작업이 끝나면 
   이벤트를 받아서 응답을 보낸다. 동시 request가 오더라도 처리가 완료될때까지 기다리지 않아도 되기 때문에 서버 부하가 적다.

npm/npx

npm: node.js의 자동화 된 의존성과 패키지 관리를 위한 패키지 매니저
1) 프로젝트를 할 때 필요한 모든 의존성 패키지는 package.json 파일 안 에서 지정할 수 있다.
2) - 한번 설치한 모듈을 계속 사용하기 때문에 업데이트 확인이 어렵다. 
   - 같은 모듈을  사용해도 프로젝트마다 다른 버전이 필요할 수 있기 때문에 버전 문제가 발생할 수 있다.
   - 모듈 변경사항이 잦은 create-react-app 같은 보일러플레이트에서는 최신버전 설치를 매번 해줘야 하기 때문에 번거롭다.  
   - npx install *** -g 사용시 글로벌 모듈을 설치할 수 있지만 업데이트 시 변동사항으로 다른 프로젝트에 영향을 줄 수 있다.

npx: npm의 모듈을 로컬에 설치했어야만 실행시킬 수 있었던 기존 문제점의 해결책/ npm 5.2 ver. 부터 npx가 기본 패키지로 제공.
       임시 모듈을 불러온다음에 프로젝트 써먹고 끝나면 없애

번외) react vs vue
react는 UI 라이브러리--> 참고 용이, 라이브러리 일부분만 가져와서 사용 편리, 리액트 만으로는 전역상태관리, 라우팅,
                                 등 지원 x 별도의 라이브러리 필요
vue는 JS 프레임워크 ---> 부분적 사용 불가, 프레임 워크가 지원해주는 문법에 따라 작성, 더 많은 기능을 디폴트로 제공 

react는 JSX 형태의 문법 사용/ JS만으로 UI 로직과 DOM 구현가능

vue는 HTML,JS,CSS 코드 영역 분리하여 사용

react는 컴포넌트의 생성 및 재사용/ 파일별로 컴포넌트 분리가능, 새로운 함수형 컴포넌트를 생산 후 props형태로 전달하거나 
          또다른 곳에서 재사용 용이
vue는  새로운 컴포넌트를 만들어 분리하기 위해서 새로운 파일을 하나 더 만들고, 그에 따라 하나의 파일에 해당하는 template,
          script, style 모두 작성해야함. 또 props를 전달하는 과정에서 해당 컴포넌트를 사용하는 모든 파일을 오가며 작성
